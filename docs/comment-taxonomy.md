# Comment Taxonomy

Use these categories to classify review comments consistently. Choose the first category that fits; avoid stacking multiple categories on one comment.

- **Technical sufficiency:** Core methods, models, or parameters are incomplete or incorrect. Use when technical execution blocks validity.
- **Reproducibility:** Missing details or artifacts needed to replicate results. Use when a competent practitioner could not rerun the analysis.
- **Unsupported assertion:** Claim lacks citation, data, or derivation. Use when text makes a statement without evidence.
- **Regulatory ambiguity:** Regulatory requirement, authority, or process is unclear or conflicting. Use when compliance path is uncertain.
- **Policy overreach:** Recommendation exceeds stated scope or authority. Use when text prescribes actions beyond evidentiary basis or mandate.
- **Missing evidence/citation:** Citation needed to support a fact, metric, precedent, or standard. Use when reference is absent or insufficient.
- **Unclear recommendation:** Action requested or implied is vague or non-specific. Use when reader cannot tell what to do.
- **Inconsistent terminology:** Terms, acronyms, or definitions are used inconsistently across the document. Use when terminology drift could confuse readers.
- **Missing operational detail:** Deployment, implementation, or interoperability detail is absent. Use when feasibility cannot be assessed.
- **Fairness/deployment burden concern:** Impact on stakeholders is uneven or onerous. Use when burden or equity implications are not addressed.
